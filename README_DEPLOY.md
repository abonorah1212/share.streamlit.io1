# 📊 Equity Intelligence — حزمة النشر (ديمو للفحص)

حزمة جاهزة لنشر التطبيق على الويب كـ**ديمو خاص** يفحصه المختصّون بأنفسهم، مع **حماية بكلمة مرور** لصون خوارزميتك.

---

## ما في الحزمة
| الملف | الغرض |
|---|---|
| `app.py` | الواجهة (مع بوابة دخول بكلمة مرور) |
| `ta_scanner.py` | محرّك التحليل |
| `requirements.txt` | المكتبات المطلوبة (مثبّتة الإصدارات) |
| `.streamlit/config.toml` | الثيم الفاتح + إعداد الخادم |
| `.streamlit/secrets.toml.example` | قالب كلمة المرور ومفاتيح API |
| `.gitignore` | يمنع رفع الأسرار |
| `render.yaml` + `runtime.txt` | نشر بنقرة على Render (مدفوع) |

---

## 🔐 أولاً: فعّل كلمة المرور (مهم)
التطبيق يعمل **مفتوحاً** إن لم تُعيّن كلمة مرور. لحماية الديمو:
- **محلياً:** انسخ `.streamlit/secrets.toml.example` إلى `.streamlit/secrets.toml` وضع رمزاً قوياً في `APP_PASSWORD`.
- **على المنصّات:** لا ترفع الملف — ضع `APP_PASSWORD` في لوحة الأسرار/البيئة (خطوات أدناه).

---

## الخيار أ — Streamlit Community Cloud (مجاني · الأسرع للاجتماع)

1. أنشئ حساباً على GitHub، وارفع ملفات الحزمة إلى مستودع جديد (خاص Private يفضّل).
2. ادخل **share.streamlit.io** بحساب GitHub → **Create app**.
3. اختر المستودع، والفرع، والملف الرئيسي = `app.py`.
4. **Advanced settings → Secrets**: الصق:
   ```
   APP_PASSWORD = "رمزك-القوي"
   ```
5. **Deploy**. خلال دقائق تحصل على رابط دائم تعطيه للمختصّين مع الرمز.

**ملاحظات:** الخطة المجانية تكفي للديمو؛ قد ينام التطبيق عند الخمول ويستيقظ عند الفتح. تظهر علامة Streamlit بسيطة.

---

## الخيار ب — Render (مدفوع · نقلة احترافية ~$7/شهر)

1. ارفع الملفات إلى مستودع GitHub.
2. ادخل **render.com** → **New → Blueprint** واختر المستودع (سيقرأ `render.yaml` تلقائياً)،
   أو **New → Web Service** يدوياً مع:
   - Build: `pip install -r requirements.txt`
   - Start: `streamlit run app.py --server.port $PORT --server.address 0.0.0.0`
3. **Environment**: أضف `APP_PASSWORD` = رمزك القوي.
4. **Create**. تحصل على رابط دائم + HTTPS تلقائي. (ابدأ بخطة Free للتجربة ثم ارفعها.)

---

## تشغيل محلّي (للتأكّد قبل النشر)
```bash
pip install -r requirements.txt
streamlit run app.py
```
ثم افتح المتصفّح على العنوان الظاهر. مع تعيين `APP_PASSWORD` ستظهر شاشة الدخول.

---

## ⚠️ قبل أن تعطي الرابط للمختصّين — اقرأ هذا
1. **ترخيص البيانات:** البيانات الحالية مجانية/متأخّرة (yfinance). عرضها لأطراف أخرى يتجاوز ترخيص الاستخدام الشخصي. للديمو المحدود المخاطرة أقل، لكن **لا تُطلقه تجارياً عاماً قبل ترخيص بيانات** (مثل Intrinio Startup الذي يتيح العرض التجاري). كن صريحاً أن البيانات تجريبية.
2. **حماية المنطق:** الكود يصل للخادم، لكن **سلوك الخوارزمية مرئي** لأي فاحص تقني عبر الواجهة. شارك الرابط مع جهات موثوقة، وفعّل كلمة المرور دائماً.
3. **إخلاء المسؤولية:** ظاهر في التطبيق («تحليل تعليمي، ليس نصيحة مرخّصة») — أبقِه.
4. **الأداء:** كل تحليل يجلب بيانات حيّة؛ مع عدّة فاحصين قد يُبطئ أو يُحظر مؤقتاً من yfinance. الكاش المدمج يخفّف ذلك؛ والبيانات المرخّصة تحلّه نهائياً.

## نصيحة للاجتماع
أعطِ المختصّين الرابط + الرمز قبل الاجتماع ليجرّبوه، أو افتحه أمامهم مباشرةً. ابدأ بحالة سهم واحد (التحليل → خطّتا المضارب/المستثمر → بطاقة العوامل الدائرية → النظرة المستقبلية) — تدفّق واحد واضح أقوى من عرض كل الميزات دفعة واحدة.
